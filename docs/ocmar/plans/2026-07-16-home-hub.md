# Session 主页化枢纽改造 实施计划 (ocdroid)

> **For agentic workers:** REQUIRED SUB-SKILL: 用 ocmar-subagent-driven-development（推荐）或 ocmar-executing-plans 按本计划逐 task 执行。步骤用 `- [ ]` 勾选追踪。

**Goal:** 把 ocdroid 从「底部 5-tab」改造为「Sessions 主页枢纽」：删底部栏、Sessions 为 startDestination、找回 0.7 服务器图标弹窗、project 列表并入主页（两可折叠 section）、files/git 经 project 行进入且标题显示当前 project、手机/平板响应式 Chat 顶左、新 tab 从右加入。

**Architecture:** hub-and-spoke NavHost。主页(Sessions)=唯一枢纽。关键并行策略见下方「并行波次」——通过**向后兼容的签名演化**（新参数带默认值）让多个屏幕 lane 在共享工作树上并发而不破坏 AppShell 编译，最后由 AppShell 集成 lane 一次性激活。

**Tech Stack:** Kotlin + Jetpack Compose + Material3 + Navigation-Compose + WindowSizeClass；JUnit4 + MockK 单测；验证 `./scripts/check.sh`。

## Global Constraints

- **UI 样式 MANDATORY**（`docs/ui-style-spec.md`）：新 overlay 走三层规则；用 `AppSectionHeader`+trailing、`ListItem`、`Dimens`（禁散落 dp）、`AppBottomSheet`/`AppConfirmDialog`/`AppFormDialog` 共享原语。
- **改动校验必做**：每个 task 收尾 `./scripts/check.sh`（compile+单测）必过；本工作区 LSP 已关，check.sh 等价手动 LSP 自检。
- **设备安全**：UI/插桩/安装**仅模拟器**（`./scripts/emulator.sh`），禁止真机（除非用户明确要求）。
- **不 commit**（ocmar 默认）；仅用户显式要求才 commit。
- **版本号**：git 派生，禁止手改 `app/build.gradle.kts`。
- **向后兼容签名**（并行关键约束）：Task 3/4/5/6 给屏幕新增的回调/参数**必须带默认值**，使现有 `AppShell.kt` 调用点继续编译；Task 7 才激活它们。

> 用户要求「尽可能安全的并行开发」。本计划按写作用域（file ownership）把工作切成**非重叠 lane**，分 3 波。同波内 lane 写不同文件，可并发。
