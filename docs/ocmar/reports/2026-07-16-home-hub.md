## 2. 方案摘要

**架构：** hub-and-spoke NavHost。Sessions=唯一枢纽；选中/新建 session→Chat；project 行→`files?workdir`/`git?workdir`；服务器弹窗→Settings。

**并行结构（C4，安全并行开发）：** admission gate（oracle `PARALLEL`，medium）+ Phase B 硬门（T0 集中 strings 后各 lane 文件集两两不相交）。3 波：
- **Wave 1（4 并发隔离 worktree，gradle-kotlin per-lane build isolation）：** laneA/T1（tab 顺序，纯单测）、laneB/T2+T3（服务器图标+主页重写）、laneC/T4（Chat 响应式）、laneD/T5+T6（Files→浏览器+Git workdir 入参）。各 lane 签名**向后兼容**（新参数带默认值），独立编译验证。
- **Wave 3（串行集成）：** T7 AppShell 枢纽（删底栏、startDestination、git 双路由、backToHome、冷启动、BackHandler 重写）。
- 串行 integrate（disjoint 文件=无冲突 rebase+ff-merge）。

**关键设计决策：**
- `backToHome()` = `setLastRoute(Sessions)` + `popBackStack(Sessions,false)` 解耦「navState 同步」与「参数化 NavController 导航」，修复 Files/Git 直接入栈不写 navState 导致 `setLastRoute(Sessions)` no-op 的 trap。
- git 单路由 `git?session={session}&workdir={workdir}`（两 nullable arg）。
- force-refresh 接 `coldStartReconnect()`（真重连，非流量快照）。
- 平板抽屉数据源镜像主页 §2a（`sessions + directorySessions` 合并）。

→ plan：`docs/ocmar/plans/2026-07-16-home-hub.md`（`a7a378`，grilling 加固 `89b150d`）
